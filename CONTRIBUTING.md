# Вклад в проект TEAM_SER_X_FEAR_BOT

Спасибо за интерес к проекту! Ниже описаны общие правила и рекомендации для совместной работы.

## Как начать

1. Форкните репозиторий.
2. Создайте ветку для своей задачи: `feature/описание`, `bugfix/описание`, `docs/описание`.
3. Вносите изменения только в своей ветке.
4. Откройте Pull Request в `SER-X-FEAR/TEAM_SER_X_FEAR_BOT`.

## Работа с issue

- Перед началом проверьте существующие issue, чтобы избежать дублирования.
- Для ошибок используйте шаблон `Bug report`.
- Для улучшений и новых функций используйте шаблон `Feature request`.

## Стиль коммитов

- Используйте понятные сообщения: `Fix bug with env load`, `Add PR template`, `Update README`.
- В описание коммита включайте краткое объяснение сделанных изменений.

## Тестирование

- Запускайте проверку синтаксиса перед коммитом:
  ```bash
  python -m compileall src
  ```
- Если добавляете новую функциональность, убедитесь, что код запускается без ошибок.

## Pull Request

- Описывайте, что изменено и почему.
- Указывайте, какие шаги выполняли для проверки.
- При необходимости добавляйте ссылки на связанные issue.

## Обратная связь

- Пожалуйста, уважайте других участников проекта.
- Если кто-то оставил комментарий, постарайтесь ответить или внести правки.

## Контакты

Если нужна помощь, создайте issue или обратитесь к администраторам организации в GitHub.
